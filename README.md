# 👻 GhostPad

Полупрозрачный блокнот, **невидимый при демонстрации экрана**.

---

## Возможности

| Функция | Описание |
|---|---|
| 🔒 Защита от захвата | Не виден в OBS, Zoom, Teams, Google Meet |
| 👻 Полупрозрачность | Стеклянный UI поверх любых окон |
| 🎚️ Прозрачность | Вертикальный ползунок в **отдельном окне справа** — следует за основным окном |
| 🗂️ Папки | Иерархическое дерево с создание/переименование/удалением, вложенностью и drag-and-drop |
| ✏️ Форматирование | Заголовки (Title/Heading/Subheading/Body), шрифты (Mono/Sans/Serif), B/I/U/S, размер, цвет, **подсветка**, выравнивание, **списки и чек-листы** |
| 🎨 Оформление | Цвет фона/акцента/тинта + пресеты (Ghost Dark, Cyberpunk, Matrix, Frosted Glass) |
| 🪟 Управление окном | Свернуть, развернуть/восстановить, полный экран (F11); размер и позиция запоминаются |
| 🖱️ Сквозной режим | Клики проходят сквозь блокнот |
| 🙈 Скрыть/Показать | Глобальная горячая клавиша |
| 💾 Авто-сохранение | Заметки, прозрачность, тема, состояние окна и дерева — всё переживает перезапуск |

## Горячие клавиши

| Клавиша | Действие |
|---|---|
| `⌘/Ctrl + Shift + T` | Включить/выключить сквозной режим |
| `⌘/Ctrl + Shift + H` | Скрыть/показать окно |
| `F11`               | Полноэкранный режим |
| `Esc`               | Закрыть открытую панель / выйти из полноэкранного |
| `⌘/Ctrl + B / I / U` | Жирный / курсив / подчёркивание |

cd ghost-pad
# 2. Установите зависимости
npm install
# 3. Запустите

## Сборка нативного приложения

Чтобы получить настоящий `.app` (macOS), `.exe` (Windows) или `.AppImage` (Linux), который можно запускать двойным кликом без терминала:

```bash
# macOS (текущая архитектура)
npm run dist:mac
# Universal для Mac
npm run dist:mac-arm  # Apple Silicon
npm run dist:mac-x64  # Intel
# Windows
npm run dist:win
# Linux
npm run dist:linux
```

Артефакты появятся в папке `release/`. Например, на macOS получится `release/GhostPad-1.2.0-arm64.dmg` и `release/mac/GhostPad.app` — `.app` можно перетащить в `/Applications`, а дальше запускать как обычное приложение.

### Иконка

Положите `icon.icns` (macOS) / `icon.ico` (Windows) / `icon.png` 512×512 (Linux) в `build/`. Если папки нет — electron-builder использует свою заглушку.

## Архитектура

```
main.js                          ← запуск, регистрация IPC и shortcuts
src/main/
  storage.js                     ← заметки + настройки + IPC
  main-window.js                 ← основной BrowserWindow, состояние, IPC
  slider-window.js               ← плавающее окно с ползунком прозрачности
index.html                       ← разметка основного окна
slider.html                      ← разметка окна-ползунка
src/renderer/
  state.js                       ← общий namespace GP
  settings.js                    ← сохранение настроек + цветовые хелперы
  theme.js                       ← темы и пресеты
  transparency.js                ← применение альфы, IPC ↔ slider-window
  window-controls.js             ← мин/макс/full + индикатор сквозного режима
  panels.js                      ← открытие/закрытие плавающей панели темы
  tree.js                        ← дерево папок и заметок, drag-and-drop, persist
  editor.js                      ← форматирование, чек-листы, paste, hotkeys
  main.js                        ← init: подгрузка настроек/заметок, биндинг событий
```

### Хранение

| Файл | Что |
|---|---|
| `userData/ghostpad-notes.json`    | `{ notes: [...], folders: [...] }`. Старый формат (плоский массив) читается, но при сохранении переводится в новый. |
| `userData/ghostpad-settings.json` | `{ window, opacity, theme, expandedFolders, selectedFolderId }`. Окно владеет ключом `window`; рендерер шлёт остальное через merge. |

### Как это работает

- **`setContentProtection(true)`** — системный API macOS/Windows, который запрещает любому ПО (Zoom, OBS, Teams…) захватывать содержимое окна. На Linux не поддерживается.
- **`setIgnoreMouseEvents(true, { forward: true })`** — сквозной режим: клики проходят насквозь.
- **`backdrop-filter: blur`** + прозрачное окно — стеклянный эффект.
- **Прозрачность** регулируется альфой CSS-фона стекла (а не `BrowserWindow.setOpacity`), поэтому текст всегда читаем и защита от захвата / сквозной режим / always-on-top не затрагиваются.
- **Окно-ползунок** — отдельный фреймлесс `BrowserWindow` с `setContentProtection(true)`, который позиционируется справа от основного окна и слушает его события `move/resize/hide/close`.
- **Дерево.** Кнопки «+» / «▤» в шапке сайдбара всегда создают в корне. Кнопки «+» и «▤» внутри строки папки — внутри неё. Удаление папки не уничтожает её содержимое: вложенные элементы поднимаются на уровень выше.
